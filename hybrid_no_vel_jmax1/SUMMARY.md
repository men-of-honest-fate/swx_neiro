## Эффект изменения clip target-весов (10 → 1) при J_max ≥ 1

Для группы **East** (где Target-W действует напрямую) изменения в пределах ±0.01–0.04 RMSLE и ±0.5 ч RMSE; направление непоследовательное:

- East/J_max: «Базовая» 0.591 → 0.598 (хуже), «Флюэс» 0.663 → 0.659 (лучше), «Обе коорд.» 0.611 → 0.620 (хуже), «Коорд+флюэс» 0.589 → 0.680 (заметно хуже).
- East/T_delta: «Обе коорд.» 11.97 → 12.08 (≈), «Коорд+флюэс» 9.99 → 10.48 (хуже на 0.5 ч); «Базовая» и «Флюэс» — без изменений.
- West/All — практически нулевая разница (|Δ| < 0.01): West использует Density-W (clip target-весов на него не влияет); All-pooled доминирован West-ом (n_west_train ≈ 3× n_east_train).

**Вывод:** опускание нижнего clip с 10 до 1 раздаёт событиям 1–10 pfu плавно растущие (вместо одинаково-низких) target-веса в East. На лучших ячейках это нейтрально или чуть ухудшает. Главные улучшения T_delta для East идут за счёт **расширения выборки** (n_test_east 3 → 11), а не за счёт clip.

## Главные выводы по порогу

1. **J_max ухудшается везде** (Δ +0.06–0.11 для West/All, +0.4–0.5 для East). Добавили события 1–10 pfu, у которых log-погрешность по самой природе больше. Метрики East при ≥10 (n=2–3) — статистически бессмысленные.
2. **T_delta — главное улучшение в East:** лучший результат — `Флюэс вместо пика`, RMSE = **8.82 ч** (с 13.5). Это побивает все предыдущие T_delta-метрики stage 0 для no_vel.
3. **Лучшее на J_max ≥ 1:** J_max — All/«Флюэс вместо пика», RMSLE = **0.729** (Linear); T_delta — All/«Координаты+флюэс», RMSE = **8.19 ч** (SVR).

## Рекомендованная конфигурация

**Подход во всех случаях:** hybrid W/E (West = Density-W с clip [0.1, 10.0]; East = Target-W α=1.5 с нижним clip J_max = 1.0), порог отбора J_max ≥ 1 pfu, признаки без `log_cme_velocity`.

### По группам West / East (модели в hybrid W/E и так раздельные)

| Группа | Задача | Набор | Модель | Метрика | 2-е место |
|---|---|---|---|---|---|
| West | J_max | Флюэс вместо пика *или* Обе координаты (tie) | **Linear** | RMSLE = **0.744** | Базовая / Linear, 0.746 |
| West | T_delta | Флюэс вместо пика | **SVR** | RMSE = **7.88 ч** | Координаты+флюэс / SVR, 8.21 ч |
| East | J_max | **Базовая** | **Forest** | RMSLE = **0.598** | Обе координаты / Boosting, 0.620 |
| East | T_delta | Флюэс вместо пика | **SVR** | RMSE = **8.82 ч** | Базовая / SVR, 10.43 ч |

**Один набор для West** — «Флюэс вместо пика» (`helio_lon`, `log_fluence`): лучший по T_delta и tie-best по J_max. Модели Linear (J_max) + SVR (T_delta).

**По East компромиссного набора нет:**
- «Базовая» (`helio_lon`, `log_goes_peak_flux`) — лучший по J_max (0.598), но проигрывает по T_delta (+1.6 ч).
- «Флюэс вместо пика» — лучший по T_delta (8.82), но проигрывает по J_max (+0.06 RMSLE).

В East локально нарушается общее правило stage 0 «флюэнс > пик» — на J_max в East пиковый поток информативнее. Если задача — одна цель, берём оптимальный набор под неё; если обе — две разные модели на разных наборах.

### По объединённой выборке All (для справки, доминирована West-ом)

| Задача | Набор | Модель | Метрика |
|---|---|---|---|
| J_max | Флюэс вместо пика | **Linear** | RMSLE = **0.729** |
| T_delta | Координаты+флюэс | **SVR** | RMSE = **8.19 ч** |

Это «один набор для обеих задач» режим, если W/E-разбиение в проде не используется. В All-pool «Флюэс вместо пика» по J_max лучше, а по T_delta — 2-е место с отрывом всего 0.12 ч.
