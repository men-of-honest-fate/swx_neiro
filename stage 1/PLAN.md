## 1. Out-of-fold prior (защита от data leakage)

**Зачем:** LSTM обучается на SC23+SC24. Prior (Этап 0) тоже обучен на SC23+SC24.
Если подать «свой» prior — LSTM увидит завышенное качество prior на обучении,
а на тесте (SC25) prior будет хуже — деградация.

**Как:**
1. Разбить SC23+SC24 (237 событий) на 5 фолдов, стратификация по W/E
2. Для каждого фолда:
   - Обучить Этап 0 на оставшихся 4 фолдах
   - Предсказать J_max_prior, T_delta_prior для отложенного фолда
3. Собрать «честные» prior-ы для всех 237 событий
4. Для SC25: использовать prior от Этапа 0, обученного на всём SC23+SC24 (как есть)

**Время:** ~5 минут (5 итераций sklearn)

---

## 2. Подготовка 5-минутных профилей

**Для каждого события:**
1. Скачать 5-мин ряд протонного потока ≥10 МэВ
2. Вырезать окно [onset, t_peak] — фаза нарастания (onset = момент превышения 1 pfu)
3. Логарифмировать: log₁₀ J(tᵢ)
4. Вычислить приращения: Δlog J(tᵢ) = log J(tᵢ) − log J(tᵢ₋₁)
5. Классифицировать событие: «чистое» или «составное»
   - Составное = есть вторая ≥M вспышка в SXR в окне [onset, t_peak]
   - Для первой итерации: обучать только на чистых, тестировать на всех

**Контроль качества:**
- Проверить пропуски в данных (GOES gaps) — интерполировать или отметить
- Проверить, что onset и t_peak согласованы с каталогом
- Удалить события с аномально длинной фазой нарастания (>20 ч, обсуждалось ранее)

## 3. Аугментация скользящим окном

**Из одного события длиной L шагов → (L − k_min) обучающих примеров**

Минимальная длина входа энкодера: k_min = 6 шагов (30 минут).

Для события с L = 36 шагами (3 часа):
- Пример 1: encoder видит шаги 1–6, target — шаги 7–36
- Пример 2: encoder видит шаги 1–7, target — шаги 8–36
- ...
- Пример 30: encoder видит шаги 1–35, target — шаг 36

**Критично:** при cross-validation все примеры из одного события — в одном фолде.

**Ожидаемый объём:** 237 событий → ~4000–5000 обучающих примеров.

## 4. Архитектура модели

```
Prior [J_max_prior, T_delta_prior]
         │
    Linear(2 → 64)
         │
         ▼ h₀
Encoder LSTM (2 слоя, hidden_size=64)
    Вход: [log J(tᵢ), Δlog J(tᵢ)] — переменная длина
         │
         ▼ h_enc ───────┐
One-shot decoder: Linear(64 → N_max)    # N_max = 250
         │
         ▼
Прогноз: [log Ĵ(t_now+5) ... log Ĵ(t_peak)]
    Длина ≤ N_max, маска на паддинг
```

**Параметры:**
- input_size = 2 (log J, Δlog J)
- hidden_size = 64
- num_layers = 2
- N_max = 250 (≈21 час, макс. фаза нарастания от 1 pfu до пика; длиннее чем от 10 pfu)
- Всего ~55 000 параметров

**Prior → h₀:**
- Linear(2 → hidden_size) → начальное скрытое состояние энкодера
- c₀ = нулевой вектор (или тоже проецировать — гиперпараметр)

---

## 5. Обучение

**Функция потерь:**

```
L = L_profile + λ · L_peak

L_profile = Σ (log Ĵ(tᵢ) − log J(tᵢ))² · mask(tᵢ) / Σ mask(tᵢ)
L_peak    = (max(output) − log J_max_real)²
```

λ = 1.0 (начальное значение, гиперпараметр)

**Оптимизация:**
- Оптимизатор: Adam
- Learning rate: 1e-3 (с подбором из {1e-3, 3e-4, 1e-4})
- Batch size: 32
- Early stopping: patience = 20–30 эпох, по валидационному MSE

**Валидация:**
- Stratified k-fold внутри SC23+SC24 (стратификация по W/E/C и log J_max)
- Все примеры из одного события — в одном фолде (без утечки)

**Гиперпараметры для подбора:**

| Параметр     | Значения           |
|------------- |--------------------|
| hidden_size  | 32, 64, 128        |
| learning rate| 1e-3, 3e-4, 1e-4   |
| dropout      | 0, 0.1, 0.2        |
| λ (вес пика) | 0.5, 1.0, 2.0      |
| k_min        | 6, 12 (30 или 60 мин) |

## 6. Бейзлайны для сравнения

### 6a. Persistence
Текущее значение потока экстраполируется вперёд: Ĵ(t) = J(t_now) = const.
Нижняя граница — если LSTM не бьёт, что-то не так.

### 6b. Степенной фит
```
log J(t) = log J₀ + α · log(t − t_onset)
```
Два параметра (J₀, α), фит через scipy.optimize.curve_fit по наблюдённым точкам.
Пересчитывается на каждом шаге (скользящий фит), аналогично LSTM.

### 6c. Этап 0 (prior only)
Просто prior: J_max = J_max_prior, T_delta = T_delta_prior.
Не обновляется по данным. Верхняя граница того, что LSTM должен побить.

## 7. Метрики

| Метрика | Формула | Что измеряет |
|---------|---------|-------------|
| RMSE(log J) | √(mean((log Ĵ − log J)²)) | Ошибка по профилю (порядки) |
| ΔJ_max | \|log Ĵ_max − log J_max\| | Ошибка пика (порядки) |
| ΔT_peak | \|T̂_peak − T_peak\| | Ошибка времени пика (часы) |
| Доля <1 порядка | % событий с ΔJ_max < 1 | Надёжность |
| Warning time | Время от первого «хорошего» прогноза до реального пика | Операциональная ценность |

**«Хороший» прогноз:** ΔJ_max < 0.5 порядка (подбирается).

**Все метрики — раздельно по W/C/E.**

**Ключевой график:** ΔJ_max vs. время после onset'а.
Ожидание: ошибка падает по мере поступления данных.
Сравнение: LSTM vs степенной фит vs persistence vs prior-only.

## 9. Решения, принятые в обсуждении

- **Порог onset:** 1 pfu (согласован с порогом prior Этапа 0).
- **Prior → h₀:** Linear(2→64) проецирует prior в начальное скрытое состояние энкодера.
- **Горизонт декодера:** только фаза нарастания (до пика), не полные 24 ч.
- **Определение пика:** max предсказанного вектора.
- **Входные каналы:** [log J(tᵢ), Δlog J(tᵢ)] — поток + градиент.
- **One-shot decoder:** Linear(h_enc → N_max) с маской на паддинг. Нет авторегрессии, нет накопления ошибки.
- **Паддинг + маска** (а не ресэмплинг к фиксированной длине).
- **Data leakage:** out-of-fold prior обязателен.
- **Бейзлайн Вейбулла/степенной фит:** нужен для обоснования сложности LSTM.

## 10. Открытые вопросы

1. **Скорость КВМ:** исключена из prior (нет в момент принятия решений). Но если КВМ наблюдается LASCO до прихода протонов — можно подать скорость как доп. канал в LSTM. Обсудить отдельно.
2. **Составные события:** пока обучаем только на чистых. Суперпозиция — Этап 3.
3. **Рентген (SXR):** второй канал для энкодера — Этап 2. После успешной реализации Этапа 1.
4. **Attention layer:** встроить в энкодер для интерпретируемости. После базовой модели.
5. **Uncertainty:** MC Dropout или quantile regression. После базовой модели.
