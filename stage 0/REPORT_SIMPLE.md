# Сравнительный анализ прогнозирования параметров СПС

## 1. Конфигурация

### Модели

| Стратегия предсказания         | Модели                                                  |
| ------------------------------ | ------------------------------------------------------- |
| Регрессия                      | Linear, RandomForest, GradientBoosting, SVR             |
| Классификация (J_max, T_delta) | LogisticRegression, RandomForest, GradientBoosting, SVC |

### Наборы признаков

| № | Название | Состав |
|---|----------|--------|
| 1 | Базовая | `helio_lon`, `log_goes_peak_flux`, `log_cme_velocity` |
| 2 | Флюэс вместо пика | `helio_lon`, `log_fluence`, `log_cme_velocity` |
| 3 | Обе координаты | `helio_lon`, `helio_lat`, `log_goes_peak_flux`, `log_cme_velocity` |
| 4 | Координаты + флюэс | `helio_lon`, `helio_lat`, `log_fluence`, `log_cme_velocity` |

**Описание признаков:**

| Признак | Описание |
|---------|----------|
| `helio_lon` | Гелиографическая долгота источника вспышки (градусы) |
| `helio_lat` | Гелиографическая широта источника вспышки (градусы) |
| `log_goes_peak_flux` | log₁₀ пикового рентгеновского потока вспышки GOES (Вт/м²) |
| `log_fluence` | log₁₀ флюэнса вспышки (интегральная характеристика, Дж/м²) |
| `log_cme_velocity` | log₁₀ скорости КВМ (км/с) из LASCO/SOHO |

### Целевые переменные

**J_max** — максимальный поток протонов СПС (pfu). Прогнозируется в логарифмическом масштабе.

**T_delta** — время от начала СПС до достижения максимума потока (часы). Прогнозируется в исходном масштабе (часы).

### Классы для классификации J_max (NOAA S-шкала)

| Класс | Диапазон J_max | Количество событий (тренировочная / тестовая) |
| ----- | -------------- | :-------------------------------------------: |
| S1–S2 | 10–100 pfu     |                   ~90 / ~25                   |
| S3    | 100–1000 pfu   |                   ~30 / ~8                    |
| S4–S5 | ≥ 1000 pfu     |                   ~18 / ~5                    |

### Классы для классификации T_delta

| Класс     | Диапазон           | Количество событий (тренировочная / тестовая) |
| --------- | ------------------ | :-------------------------------------------: |
| Быстрые   | T_delta < 8 ч      |                   48 / ~18                    |
| Умеренные | 8 ≤ T_delta < 20 ч |                   54 / ~10                    |
| Медленные | T_delta ≥ 20 ч     |                    36 / ~9                    |

### Метрики

| Задача | Метрика | Формула / описание |
|--------|---------|-------------------|
| Регрессия J_max | RMSLE $\log_{10}$ | $\sqrt{\dfrac{1}{n}\sum_{i=1}^{n}\!\left(\log_{10}\hat{y}_i - \log_{10} y_i\right)^2}$ — ниже = лучше |
| Регрессия J_max | $R^2_{\log}$ | Коэффициент детерминации в $\log_{10}$-пространстве |
| Регрессия J_max | $\rho_s$ | Ранговая корреляция Спирмена |
| Регрессия T_delta | RMSE | $\sqrt{\dfrac{1}{n}\sum_{i=1}^{n}(\hat{y}_i - y_i)^2}$ в часах — ниже = лучше |
| Классификация | Log Loss | $-\dfrac{1}{n}\sum_{i=1}^{n}\sum_{k} y_{ik}\ln p_{ik}$ — ниже = лучше |
| Классификация | Accuracy | Доля правильно классифицированных событий |
| Классификация | AUC macro | Macro-averaged ROC AUC по всем классам OvR — выше = лучше |

## 2. Результаты

### 2.1 Регрессия J_max

**Метрика:** RMSLE log₁₀ (ниже = лучше)

| Набор признаков       | Test RMSLE | R²_log | ρ_s  | Лучшая модель |
| --------------------- | :--------: | :----: | :--: | :-----------: |
| Базовая               |   0.691    |  0.14  | 0.54 |    Linear     |
| **Флюэс вместо пика** | **0.670**  |  0.20  | 0.59 |    Linear     |
| Обе координаты        |   0.691    |  0.15  | 0.54 |    Linear     |
| Координаты + флюэс    |   0.707    |  0.10  | 0.50 |    Linear     |

**Выводы:**
- Набор «Флюэс вместо пика» обеспечивает наименьшее RMSLE = 0.670 (ошибка ≈ ×4.7 по потоку).
- Добавление широты (`helio_lat`) к базовому набору не улучшает результат — при объёме выборки 138 событий широтная информация не улучшает обобщение.
- Замена пика на флюэнс при наличии широты («Координаты + флюэс») даёт ухудшение (0.707)
- Linear-регрессия стабильно превосходит RandomForest, Boosting и SVR во всех 4 наборах признаков — задача линейна в логарифмическом пространстве.

### 2.2 Регрессия T_delta

**Метрика:** RMSE (часы)

| Набор признаков | Test RMSE (ч) | R² | ρ_s | Лучшая модель |
|----------------|:-------------:|:--:|:---:|:-------------:|
| **Базовая** | **14.35** | 0.14 | 0.18 | Linear |
| Флюэс вместо пика | 14.44 | 0.13 | 0.11 | Linear |

**Выводы:**

- Все значения R² ≤ 0.14 и ρ_s ≤ 0.18 подтверждают принципиальную ограниченность прогнозируемости T_delta доступными признаками.
- Linear неизменно лучшая модель — нелинейные паттерны в данных незначительны.

### 2.3 Классификация J_max (NOAA S-шкала)

**Классы:** S1–S2 (10–100 pfu), S3 (100–1000 pfu), S4–S5 (≥1000 pfu)
**Метрика:** Log Loss (ниже = лучше), AUC (выше = лучше)

| Набор признаков | Test Log Loss | Test Accuracy | Test AUC | Лучшая модель |
|----------------|:-------------:|:-------------:|:--------:|:-------------:|
| Базовая | 0.784 | 65% | 0.722 | Forest |
| **Флюэс вместо пика** | 0.755 | 65% | 0.760 | Forest |
| Обе координаты | 0.794 | 65% | 0.719 | Forest |
| **Координаты + флюэс** | **0.757** | 65% | **0.778** | Forest |

**Выводы:**
- Флюэнс последовательно улучшает классификацию S-классов по сравнению с пиковым потоком (Log Loss снижается с 0.784 до 0.755–0.757).
- RandomForest является лучшей моделью для всех 4 наборов — нелинейные взаимодействия между признаками значимы для разделения S-классов.
- Accuracy = 65% во всех конфигурациях определяется дисбалансом тестовой выборки (66% событий класса S1–S2).

### 2.4 Классификация T_delta

**Классы:** Быстрые (< 8 ч), Умеренные (8–20 ч), Медленные (≥ 20 ч)
**Метрика:** Log Loss (ниже = лучше), AUC (выше = лучше)

| Набор признаков | Test Log Loss | Test Accuracy | Test AUC | Лучшая модель |
|----------------|:-------------:|:-------------:|:--------:|:-------------:|
| **Базовая** | **0.975** | 48% | **0.730** | Forest |
| Флюэс вместо пика | 0.999 | 55% | 0.601 | SVC |
| Обе координаты | 1.033 | 55% | 0.619 | SVC |
| Координаты + флюэс | 1.023 | 55% | 0.590 | SVC |

**Выводы:**
- «Базовая» конфигурация (гелиодолгота + пиковый поток + скорость КВМ) обеспечивает наилучший AUC = 0.730 для классификации T_delta. Геометрическая и кинематическая информация о КВМ наиболее информативна для разделения классов скорости прихода.
- Замена пикового потока на флюэнс ухудшает AUC (0.730 → 0.601): флюэнс характеризует энерговыделение вспышки, но не несёт прямой информации о транспорте частиц.
- Добавление широты (`helio_lat`) не улучшает классификацию T_delta ни в одной из конфигураций.
- Относительно высокий тестовый AUC = 0.730 при Log Loss = 0.975 и Accuracy = 48% объясняется сильным дисбалансом класса «Быстрые» в тестовой выборке SC25 (медиана T_delta_test ≈ 5 ч vs. медиана T_delta_train ≈ 10 ч).

## 4. Выводы

1. **Флюэнс превосходит пиковый поток для J_max** во всех наборах признаков. Для регрессии «Флюэс вместо пика» даёт RMSLE = 0.670 (лучший результат). Для классификации флюэнс снижает Log Loss с 0.784 до 0.755 и повышает AUC с 0.722 до 0.760.

2. **Гелиоширота (`helio_lat`) не улучшает прогноз** ни для одной задачи. Для регрессии J_max её добавление нейтрально; для T_delta — ухудшает (RMSE растёт с 14.35 до 14.99–15.06 ч); для классификации J_max — умеренное улучшение AUC только при одновременном добавлении флюэнса (AUC = 0.778 для «Координаты + флюэс»).

3. Классификация T_delta осуществима: AUC = 0.730 для задачи разделения на классы Быстрые / Умеренные / Медленные при базовой конфигурации существенно выше случайного (0.5). Это качественно лучше точечной регрессии T_delta (R² ≤ 0.14), что указывает на то, что задача принадлежности к классу скорости более решаема, чем точное восстановление времени прихода.

4. Линейная регрессия доминирует для точечных прогнозов** J_max и T_delta. **RandomForest** лучший для задач классификации. Паттерн устойчив по всем 4 наборам признаков.
